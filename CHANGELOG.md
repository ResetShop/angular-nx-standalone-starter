# Changelog

All notable changes to this starter are documented here. Forks should read this file before every upstream merge — it is the single source of truth for what changed upstream and whether any of those changes affect fork-owned code or require migration work.

The format is based on [Keep a Changelog 1.1.0](https://keepachangelog.com/en/1.1.0/). Versioning cadence: **every fork-visible breaking change cuts a new version**; non-breaking changes accumulate under `## [Unreleased]` until the next breaking change forces a new release. A "fork-visible breaking change" is anything that requires a fork to take action during a merge — removed files, renamed APIs, changed schemas, restructured directories, or modified contracts. Pre-1.0 versions use the `1.0.0-beta.N` numbering convention while the fork-distribution model is being stabilized.

### Added

- **`docs/thermo-nuclear-skill-evaluation.md` — evaluation of Cursor's `/thermo-nuclear-code-quality-review` skill vs. our `code-reviewer` agent.** Spike report (the agent/performance slice of [#378](https://github.com/ResetShop/angular-nx-standalone-starter/issues/378)). Installed the skill via `npx skills add https://github.com/cursor/plugins --skill thermo-nuclear-code-quality-review` (vercel-labs `skills` CLI; installs to a global agents dir, not the repo) and ran its prompt against two representative real files (`users.store.ts`, `user-management.repository.ts`). **Key finding:** the skill is **prompt-only** (a strict-maintainability review prompt — no scripts/executable code, so negligible supply-chain surface). It is a strong _structural-simplification idea generator_ (surfaced duplicated mutation skeletons, repeated projection literals, redundant pre-`SELECT` round-trips) but is **context-blind** — it would contradict project conventions (mandated `rxMethod`, the two-`withMethods` store split, `find*()` naming, file-local `Projection` types) on less-compliant code, with a ~30–40% false-positive/anti-recommendation rate. **Recommendation: AUGMENT** — wire it as an optional pre-review structural pass **gated behind** `code-reviewer` (which loads its full reference set and runs CI), and only after amending it to load `CLAUDE.md` + `.claude/references/` first; **not replace** (no project hard-constraint / security / test / docs / CI coverage) and **not reject** (real, repeatable structural value). A follow-up implementation issue is **recommended** (scope drafted in the report; intentionally left for a maintainer to file). Documentation-only; no runtime, schema, or fork-API impact. ([#409](https://github.com/ResetShop/angular-nx-standalone-starter/issues/409))

- **`npm run ci:verify` — cache-aware CI for the agent inner loop.** New `package.json` script that runs the identical task set as `ci` (`stylelint`, `lint`, `typecheck`, then `test`, `test-integration`, `build`, `build-storybook`) but **without** `--skip-nx-cache`, so repeated runs on a largely-unchanged tree restore from the Nx local cache (and, once remote caching lands per [#406](https://github.com/ResetShop/angular-nx-standalone-starter/issues/406), the Nx Cloud remote cache) instead of forcing a full cold rebuild. The cold `npm run ci` is **unchanged** and remains the authoritative final gate — it keeps `--skip-nx-cache` to guarantee correctness independent of cache state. This removes the redundant cold-rebuild tax from the 2–3 intermediate CI runs the `/issue-workflow` skill performs per issue (Phase 4 orchestrator, inside `code-reviewer`, Phase 5 post-fix) while preserving a strict cold run for the release gate. - **Decouple user-management from authentication-table writes (internal backend refactor).** `DrizzleUserManagementRepository` (`apps/reference-app/src/api/modules/user/user-management.repository.ts`) no longer writes to the `authentication` table: the `tx.insert(authentication)` inside `create()` and the entire `updatePasswordAndMustChange()` method are gone, and the `@schema/authentication` import is removed. The service gains `authRepository` and `userRoleRepository` constructor dependencies, resolved by Awilix from their existing cradle keys (no `container.ts` registration change). At the repo layer `create()` drops both its password fields and its role writes: `CreateUserWithHashedPasswordParams` is removed from `user-management/interfaces.ts` and replaced by the identity-only `CreateUserIdentityParams` (`{ email, firstName, lastName }` — no `roleIds`); `create(params, tx?)` (no `actorId`) and `runInTransaction` are added to the `UserManagementRepository` interface, and `UserRoleRepository.replaceUserRoles` gains an optional `tx` so it can join the creation transaction. **Request/response shapes, URLs, and frontend code are unchanged.** One status-code change: `POST /api/user` with non-existent `roleIds` now returns a documented `400` (`Roles not found`) — the same validation the `PUT /api/user/{id}/roles` endpoint already enforced — instead of the previous unhandled foreign-key error; the `201` success path and `POST /api/user/{id}/reset-password` are unchanged. **Migration:** forks that hand-instantiate `new UserManagementService({ … })` outside the container must add an `authRepository: AuthenticationRepository` constructor dep; forks that called `UserManagementRepository.updatePasswordAndMustChange(...)` or built `CreateUserWithHashedPasswordParams` directly must switch to `AuthenticationRepository.setPassword(...)` / `createInitialPassword(...)` and the identity-only `CreateUserIdentityParams` (`{ email, firstName, lastName }`); forks that hand-instantiate `new UserManagementService({ … })` must also add a `userRoleRepository: UserRoleRepository` constructor dep. - **Sidebar nav flicker during logout** — `provideAuth()` (`apps/reference-app/src/app/providers/auth/auth.provider.ts`) now accepts opt-in features in the `provideRouter`/`provideHttpClient` style. The new `withNavigationPermissionCheck()` feature wires the navigation library's `NAVIGATION_PERMISSION_CHECK` token through `AuthStore.currentUser`, and **returns `true` for any permission when no user is logged in**. ## [1.0.0-beta.0] — 2026-04-06
